# Story 8-2: Stacks Tab UI -- Browse & Preview

## Status: done
## Epic: Epic 8 -- Stack Browsing & Placement

## Intent
**Epic Handle:** "Pre-wired building blocks, one drag away"
**Story Handle:** "This story fills the empty Stacks tab with browsable cards showing what's inside each pre-wired pattern"

## Overview

As a user,
I want to browse pre-composed technology stacks in the Stacks tab with cards showing components, connections, and trade-off profiles,
So that I can evaluate which architectural pattern fits my needs before placing it.

**FRs:** FR64, FR65
**Size:** MEDIUM (4 pts) -- 4 tasks, ~18 subtasks, 5 files
**Risk Flags:** PURE_COMPONENT, ERROR_RESILIENCE
**Depends on:** Story 8-1 (stack data model)

## Acceptance Criteria

**AC-1: Stacks Tab Populated (FR64)**
Given the Three-Tab Toolbox,
When the user clicks the Stacks tab,
Then pre-composed technology stacks are listed as browsable cards.

**AC-2: Stack Card Content (FR65)**
Given a stack card,
When displayed,
Then it shows: stack name, description, included components (icons + names), connection count, and trade-off profile summary (mini category bars).

**AC-3: Stack Detail Expansion (FR65)**
Given a stack card,
When clicked or expanded,
Then a detail view shows: full component list with variants, connection descriptions, and trade-off profile per category.

**AC-4: Empty State**
Given no stacks loaded from the library,
When viewing the Stacks tab,
Then a helpful empty state message is shown (not a blank tab).

## Architectural Acceptance Criteria

**AC-ARCH-LOC-1:** StackCard component at `src/components/toolbox/StackCard.tsx`
**AC-ARCH-LOC-2:** StacksTab at `src/components/toolbox/StacksTab.tsx` (modify existing placeholder)
**AC-ARCH-PATTERN-1:** StackCard is a pure presentational component -- receives StackDefinition as prop
**AC-ARCH-PATTERN-2:** StacksTab reads from componentLibrary.getStacks() -- no direct Firestore calls
**AC-ARCH-PATTERN-3:** Trade-off profile displayed as mini CategoryBar components (reuse from dashboard)
**AC-ARCH-PATTERN-4:** Stack cards are draggable -- drag data includes stack ID (placement handled in Story 8-3)
**AC-ARCH-NO-1:** No canvas placement logic in this story -- browse only
**AC-ARCH-NO-2:** No modifications to architectureStore -- browsing is read-only from library

## Dev Notes

**Existing StacksTab:** There's already a placeholder Stacks tab in the toolbox. This story replaces placeholder content with real stack browsing.

**Drag data format:** Use HTML5 drag events with `dataTransfer.setData('application/archie-stack', stackId)`. The canvas drop handler (Story 8-3) reads this data type.

**Trade-off profile mini bars:** Reuse the CategoryBar component from the dashboard but in a compact size. Pass the stack's pre-computed tradeOffProfile scores.
