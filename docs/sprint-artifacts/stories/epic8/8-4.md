# Story 8-4: E2E -- Stack Browsing & Placement Flow

## Status: drafted
## Epic: Epic 8 -- Stack Browsing & Placement

## Intent
**Epic Handle:** "Pre-wired building blocks, one drag away"
**Story Handle:** "This story validates the complete stack browsing and placement experience end-to-end"

## Overview

As a developer,
I want E2E tests covering the stack browsing and placement flow,
So that regressions in stack cards, drag-and-drop placement, and post-placement editing are caught automatically.

**FRs:** Validates FR64-FR68 end-to-end
**Size:** SMALL (2 pts) -- 2 tasks, ~10 subtasks, 2 files
**Risk Flags:** E2E_TESTING
**Depends on:** Story 8-2 (browse UI), Story 8-3 (placement)

## Acceptance Criteria

**AC-1: Stack Browsing**
Given authenticated user,
When E2E test navigates to Stacks tab,
Then stack cards are visible with names, component lists, and trade-off profiles.

**AC-2: Stack Placement**
Given a stack card in the Stacks tab,
When E2E test drags it onto the canvas,
Then all components appear on canvas with connections.

**AC-3: Post-Placement Editing**
Given stack components placed on canvas,
When E2E test changes a component's configuration variant,
Then metrics recalculate and heatmap updates.

## Architectural Acceptance Criteria

**AC-ARCH-LOC-1:** Spec at `tests/e2e/stack-browsing.spec.ts`
**AC-ARCH-PATTERN-1:** SCREENSHOT_DIR constant, data-testid selectors, getByRole fallback
**AC-ARCH-PATTERN-2:** Auth pre-loaded from storageState
**AC-ARCH-PATTERN-3:** Screenshots to `test-results/stack-browsing/`
**AC-ARCH-NO-1:** No waitForTimeout > 1000ms
**AC-ARCH-NO-2:** No browser.newContext() for authenticated tests

## Dev Notes

**Drag-and-drop in Playwright:** Use `page.locator('[data-testid="stack-card-{id}"]').dragTo(page.locator('[data-testid="canvas-area"]'))`. Playwright supports drag-and-drop natively.

**Stack data dependency:** These tests require stack definitions in the Firestore emulator seed data. Ensure the test environment has at least one stack loaded.
