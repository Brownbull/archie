## Acceptance Criteria

**AC-1: Stack Browsing**
Given authenticated user,
When E2E test navigates to Stacks tab,
Then stack cards are visible with names, component lists, and trade-off profiles.

**AC-2: Stack Placement**
Given a stack card in the Stacks tab,
When E2E test drags it onto the canvas,
Then all components appear on canvas with connections.

**AC-3: Post-Placement Editing**
Given stack components placed on canvas,
When E2E test changes a component's configuration variant,
Then metrics recalculate and heatmap updates.

## Dev Notes

**Drag-and-drop in Playwright:** Use `page.locator('[data-testid="stack-card-{id}"]').dragTo(page.locator('[data-testid="canvas-area"]'))`. Playwright supports drag-and-drop natively.

**Stack data dependency:** These tests require stack definitions in the Firestore emulator seed data. Ensure the test environment has at least one stack loaded.

**Implementation (2026-03-13):** Used synthetic DragEvent approach (matching `dragComponentToCanvas`) with `application/archie-stack` MIME type. Added `waitForStacksTab` and `dragStackToCanvas` helpers to canvas-helpers.ts. Tests skip gracefully when no stack seed data is present.
