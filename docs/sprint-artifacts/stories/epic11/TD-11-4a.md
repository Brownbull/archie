## Story

As a **developer**, I want **unit tests covering the per-component failure override logic and directional accuracy pins**, so that **the behavioral code added in 11-4 is verified and data regressions are caught at CI time**.

## Acceptance Criteria

**AC-1: RecalculationService Override Logic Test**
Given a component with `failureResponses` data and a component without,
When `recalculationService.run()` is called with an active failure preset,
Then the component with specific data uses its `failureResponses[presetId]` modifiers, and the component without falls back to the global `failureModifiers`.

**AC-2: Directional Accuracy Pins**
Given all 8 new components with `failureResponses`,
When specific preset+metric combinations are checked,
Then values match expected directional ranges (e.g., LLM Gateway `failure-traffic-spike` → `cost-per-inference` ≤ 0.4; Data Lake `failure-data-corruption` → `data-durability` ≤ 0.4).
