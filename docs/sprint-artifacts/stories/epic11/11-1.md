## Intent
**Epic Handle:** "More building blocks for more architectures"

## Overview

As a user,
I want LLM Gateway, Vector Database, Serverless Compute, and ETL Pipeline components available in the toolbox,
So that I can model AI, serverless, and data engineering architectures.

**FRs:** FR95
**Size:** MEDIUM (4 pts) -- 3 tasks, 14 subtasks, 4 files (data-only)
**Risk Flags:** DATA_PIPELINE, INPUT_SANITIZATION
**Depends on:** None -- data files only, no code changes

## Acceptance Criteria

**AC-1: 4 Component YAML Files Created (FR95)**
Given the existing 10 components in the library,
When 4 new component YAML files are added,
Then the toolbox shows 14 total components, each browsable and placeable.

**AC-2: Schema Compliance (NFR26)**
Given each new component YAML file,
When validated against `ComponentYamlSchema`,
Then all 4 pass without errors. Every field matches the Zod schema exactly.

**AC-3: Configuration Variants**
Given each new component,
When its config variants are reviewed,
Then each has 2-3 meaningful configuration variants with distinct metric trade-offs and code snippets.

**AC-4: Metric Coverage**
Given each new component,
When its base_metrics are reviewed,
Then each has metrics across at least 4 of the 7 existing categories (performance, reliability, scalability, security, operational-complexity, cost-efficiency, developer-experience).

**AC-5: Connection Properties**
Given each new component,
When its connection_properties are reviewed,
Then each defines protocol, communication_patterns, typical_latency, and co_location_potential.

**AC-6: Metric Explanations**
Given each new component variant,
When metric_explanations are reviewed,
Then each variant explains at least its 2 most distinctive metrics with reason + contributing_factors.
