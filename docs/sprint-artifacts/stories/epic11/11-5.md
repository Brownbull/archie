## Dev Notes

**Blueprint YAML format** follows existing convention from `src/data/blueprints/whatsapp-messaging.yaml`:
```yaml
id: ai-agent-orchestration
name: "AI Agent Orchestration"
description: "Multi-model AI agent pipeline with vector retrieval and task queuing..."
skeleton:
  schema_version: "1.0.0"
  name: "AI Agent Orchestration"
  nodes:
    - id: ai-llm
      component_id: llm-gateway
      config_variant_id: multi-model-router
      position:
        x: 608
        y: 192
    # ...
  edges:
    - id: e1
      source: ai-llm
      target: ai-vectordb
```

**Domain rationale:**
- Social Media Feed: shows Graph DB + real-time (WebSocket + Kafka) -- relationship-heavy, read-optimized
- AI Agent: shows LLM Gateway + Vector DB -- retrieval-augmented generation pattern
- Data Analytics: shows ETL + Data Lake + monitoring -- batch/stream hybrid
- Video Streaming: shows CDN + caching + queuing -- delivery-optimized
- FinTech: shows Payment Gateway + SQL + caching -- compliance + reliability focused
- Video Streaming: serverless replaces Kafka per AC-2 (must use at least one 11-1/11-2 component)
