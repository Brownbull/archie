# Story 11-6: E2E -- New Components + Blueprints

## Status: drafted
## Epic: Epic 11 -- Expanded Content Library

## Intent
**Epic Handle:** "More building blocks for more architectures"
**Story Handle:** "This story proves the expanded content works end-to-end -- V6 semantic verification across all new data"

## Overview

As a user,
I want end-to-end proof that the 8 new components load correctly, 5 new blueprints render correctly, cost-efficiency metrics appear in the dashboard, and demand/failure scenarios affect new components,
So that I can trust the expanded content works as intended.

**FRs:** FR95, FR96, FR97, FR98, FR99 (cross-cutting validation)
**Size:** SMALL (2 pts) -- 3 tasks, 10 subtasks, 2 files
**Risk Flags:** E2E_TESTING
**Depends on:** Stories 11-1, 11-2, 11-3, 11-4, 11-5 (all Epic 11 content stories complete)

## Acceptance Criteria

**AC-1: New Components Load in Toolbox**
Given the 18 total components in the library,
When the toolbox renders,
Then all 18 components are visible, browsable by category, and placeable on the canvas.

**AC-2: New Blueprints Load on Canvas**
Given the 15 total blueprints,
When the user loads "AI Agent Orchestration" blueprint,
Then all referenced components appear with correct positions, connections, and inline metrics visible.

**AC-3: Cost-Efficiency Metric Visible (V6: Semantic Verification)**
Given a component placed on the canvas with cost-efficiency weight set to 1.0,
When inline metrics render,
Then at least one card shows "Cost" as one of the top 2 displayed metric categories. The value is a number between 1-10.

**AC-4: Demand Scenario Affects New Components (V6)**
Given an AI Agent Orchestration blueprint loaded and "Traffic Peak" scenario selected,
When the canvas renders,
Then the LLM Gateway component shows more degradation than the Redis Cache component (directional correctness).

**AC-5: Failure Scenario Affects New Components (V6)**
Given new components on canvas and "Database Failure" scenario selected,
When the canvas renders,
Then components with data storage category (Vector DB, Graph DB, Data Lake) show significant degradation.

**V6 compliance:** Every assertion verifies CONTENT CORRECTNESS, not just DOM PRESENCE. Examples:
- GOOD: `expect(llmGatewayScore).toBeLessThan(redisCacheScore)` (directional)
- BAD: `expect(page.locator('[data-testid="metric-bar"]')).toBeVisible()` (presence only)

**Selector priority:** `data-testid` > `getByRole` > scoped locator > bare text (last resort)
