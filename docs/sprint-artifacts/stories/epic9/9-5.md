## Overview

As a user,
I want end-to-end proof that demand simulation changes my architecture's visual feedback meaningfully,
So that I can trust the simulation reflects real demand pressure, not just arbitrary metric changes.

## Acceptance Criteria

**AC-1: Scenario Selection Changes Heatmap**
Given an architecture with 3+ components on the canvas,
When "Traffic Peak" scenario is selected,
Then at least one component's heatmap status visibly shifts (e.g., green → yellow or yellow → red).

**AC-2: Weak Components Degrade More Than Strong (V6: Semantic Verification)**
Given components with different demand resilience (e.g., PostgreSQL vs Nginx under Traffic Peak),
When the scenario is active,
Then PostgreSQL's visible metrics degrade more than Nginx's (verifying MEANING, not just that something changed).

**AC-3: Pathway Suggestions Respond to Demand**
Given an active demand scenario,
When pathway suggestions are visible,
Then at least one suggestion is demand-relevant (e.g., "Add Redis Cache" appears or ranks higher under Traffic Peak).

**AC-4: Scenario Deselection Restores Baseline**
Given an active scenario that shifted the heatmap,
When the user selects "No Scenario",
Then the heatmap reverts to its pre-scenario state.

**AC-5: YAML Round-Trip Preserves Scenario**
Given an active scenario,
When the architecture is exported and re-imported,
Then the same scenario is active after import.
