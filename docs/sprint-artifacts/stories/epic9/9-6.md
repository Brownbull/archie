## Overview

As a user,
I want to see particle animations on connections reflecting connection health and smooth transitions on metric bars when values change,
So that I get visual feedback showing how my architecture responds to scenario and weight changes.

## Acceptance Criteria

**AC-1: Particle Animation on Connections (FR89)**
Given components connected on the canvas,
When the architecture has weighted scores computed,
Then each connection renders particles whose density (count per edge) reflects the connection health score. Higher health = denser particles; lower health = sparse particles.

**AC-2: Density Maps to Health Score (FR89)**
Given a connection with health score 8.0 (healthy) and another with 3.0 (bottleneck),
When both render particles,
Then the healthy connection shows visibly more particles than the bottleneck connection. (Directional, not exact count.)

**AC-3: Particle Speed = Data Direction (FR89)**
Given a connection with defined source and target,
When particles animate,
Then particles flow from source to target, providing directional visual feedback.

**AC-4: Animated Metric Bar Transitions (FR90)**
Given inline metric bars on canvas cards (from Story 10-1),
When a scenario is selected or weight profile changes cause metric values to shift,
Then the metric bars animate smoothly from the old value to the new value (not an instant jump).

**AC-5: Toggle-able Animation (NFR23)**
Given the preferences panel,
When the user toggles animations off,
Then all particle animations stop and metric bar transitions become instant. Canvas performance returns to non-animated baseline.

**AC-6: Performance Budget (NFR23)**
Given 15 components with 20 connections on the canvas with animations enabled,
When the canvas renders,
Then the animation loop maintains <16ms per frame (60fps target). No frame drops below 30fps.

## Dev Notes

**Animation = score visualization, NOT traffic flow.** Particle density reflects connection health quality (high health = dense particles, bottleneck = sparse). This is a visual metaphor for "data flows freely through healthy connections."

**No animation framework.** Use raw `requestAnimationFrame` + CSS transitions. This keeps bundle size small and avoids dependency risk.

**Performance budget:** The RAF loop should batch particle position updates. If >20 edges exist, consider reducing particle count per edge proportionally to stay within frame budget.
