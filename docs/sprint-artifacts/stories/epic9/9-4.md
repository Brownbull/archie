# Story 9-4: Scenario Selector UI & Store Integration

## Status: done
## Epic: Epic 9 -- Demand Simulation

## Intent
**Epic Handle:** "See your architecture under pressure"
**Story Handle:** "This story gives users a single dropdown that reshapes the entire canvas — select a scenario, see your architecture react"

## Overview

As a user,
I want to select a demand scenario from a dropdown and see my architecture's heatmap, metrics, and pathway suggestions update instantly,
So that I understand how my architecture holds up under different operating conditions.

**FRs:** FR83, FR84, FR85
**Size:** LARGE (6 pts) -- 5 tasks, ~22 subtasks, 8 files
**Risk Flags:** CROSS_STORE, PURE_COMPONENT, E2E_TESTING
**Depends on:** Story 9-3 (demand engine)

## Acceptance Criteria

**AC-1: Scenario Dropdown (FR83)**
Given the canvas toolbar,
When the scenario selector is visible,
Then it shows "No Scenario" (default) plus 6 preset scenarios with name and icon. Selecting a scenario activates it immediately.

**AC-2: Heatmap Updates Under Demand (FR84)**
Given an active scenario and components on the canvas,
When the scenario is selected,
Then the heatmap recalculates using demand-adjusted metrics (not base metrics). Components that degrade under the scenario shift toward warning/bottleneck status.

**AC-3: Dashboard Updates (FR84)**
Given an active scenario,
When viewing the architecture dashboard,
Then category scores reflect demand-adjusted metrics. An indicator shows the active scenario name.

**AC-4: Pathway Suggestions Respond to Demand (FR85)**
Given an active scenario,
When pathway suggestions are computed,
Then suggestions account for demand-adjusted metrics (e.g., under Traffic Peak, a cache component ranks higher because databases degrade more).

**AC-5: Scenario Deselection**
Given an active scenario,
When the user selects "No Scenario" from the dropdown,
Then all metrics revert to base values and the heatmap/dashboard/pathway suggestions recalculate without demand modifiers.

**AC-6: YAML Export/Import (FR85)**
Given an active scenario when exporting YAML,
When the architecture is exported,
Then the active scenario ID is included in the YAML header. On import, the scenario is restored if it matches a known preset.

**AC-7: Demand Indicator on Canvas (FR83)**
Given an active scenario,
When viewing the canvas,
Then a visible indicator (badge/banner) shows the active scenario name, distinguishing demand-adjusted state from baseline.

## Dev Notes

**Recalculation pipeline order (updated):**
```
Base metrics (component library)
    ↓
Config variant overrides (existing)
    ↓
Demand modifiers (NEW — this story)
    ↓
Weighted scoring (existing priority weights)
    ↓
Heatmap + Dashboard + Pathway guidance (existing)
```

**V2 alignment:** When a scenario is active, the canvas indicator should make it clear that metrics are demand-adjusted, not baseline. Users should understand they're seeing a simulation, not ground truth.

**Performance:** Demand engine is called as part of recalculation — no separate render cycle. Total recalculation budget remains <200ms (NFR12).
