# Story 5-3: Priority Slider UI -- Controls & Dashboard Integration

## Status: done
## Epic: Epic 5 -- Priority Scoring

## Intent
**Epic Handle:** "Your priorities reshape the scoreboard"
**Story Handle:** "This story builds the physical faders on the mixing board by giving the user interactive slider controls"

## Overview

As a user,
I want to adjust 7 category weight sliders and see my architecture re-scored in real time,
So that I can explore how different priorities change the picture.

**FRs:** FR44, FR48, FR49, NFR12, NFR19
**Size:** MEDIUM (4 pts) -- 5 tasks, ~21 subtasks, 6 files
**Risk Flags:** PURE_COMPONENT, CROSS_STORE
**Depends on:** Story 5-1 (types), Story 5-2 (store actions + weighted engine)

## Acceptance Criteria

**AC-1: Category Weight Sliders (FR44)**
Given the dashboard expanded overlay is open,
When viewing categories,
Then each of the 7 categories has a weight slider with range 0.1-1.0, step 0.1, default 1.0.

**AC-2: Real-time Re-scoring (FR45, FR46, FR47, NFR12)**
Given I drag a slider and release it,
When the value changes,
Then dashboard scores, heatmap colors, and tier update within 200ms.

**AC-3: Reset Weights (FR48)**
Given sliders adjusted to non-default values,
When I click "Reset Weights",
Then all sliders snap to 1.0 and scores revert to unweighted values.

**AC-4: Dual Score Display (FR49)**
Given weights adjusted (any weight differs from 1.0),
When viewing dashboard,
Then both weighted and unweighted aggregate scores shown side by side.
When all at default, only single score shown.

**AC-5: Value Clamping (NFR19)**
Given a slider value set,
When it would fall outside [0.1, 1.0],
Then the value is clamped to valid range in 0.1 steps.

## Architectural Acceptance Criteria

**AC-ARCH-LOC-1:** WeightSliders component at `src/components/dashboard/WeightSliders.tsx`
**AC-ARCH-PATTERN-1:** Reads store via selector `useArchitectureStore((s) => s.weightProfile)` -- not entire store
**AC-ARCH-PATTERN-2:** Dispatches `setWeightAndRecalculate` -- does NOT import engine functions
**AC-ARCH-PATTERN-3:** Debounce onChange at 100ms to prevent recalculation storms
**AC-ARCH-PATTERN-4:** Reset button calls `setWeightProfile(DEFAULT_WEIGHT_PROFILE)`
**AC-ARCH-PATTERN-5:** Sliders in DashboardOverlay (expanded), not compact DashboardPanel
**AC-ARCH-PATTERN-6:** Compact DashboardPanel shows weight badge on CategoryBar when weight != 1.0
**AC-ARCH-NO-1:** No engine imports in WeightSliders component
**AC-ARCH-NO-2:** No useState for slider values -- source of truth is architectureStore.weightProfile
**AC-ARCH-NO-3:** No slider controls in compact DashboardPanel
