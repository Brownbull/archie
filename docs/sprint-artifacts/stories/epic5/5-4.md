# Story 5-4: YAML Weight Profile Export/Import Round-Trip

## Status: review
## Epic: Epic 5 -- Priority Scoring

## Intent
**Epic Handle:** "Your priorities reshape the scoreboard"
**Story Handle:** "This story saves and restores the mixing board preset by ensuring weight profiles round-trip through YAML"

## Overview

As a user,
I want my priority slider positions saved when I export YAML and restored when I import,
So that my priority configuration is portable and shareable alongside my architecture.

**FRs:** FR50, FR72, FR71 (end-to-end)
**Size:** SMALL (3 pts) -- 3 tasks, ~16 subtasks, 5 files
**Risk Flags:** DATA_PIPELINE, ERROR_RESILIENCE
**Depends on:** Story 5-1 (schema), Story 5-2 (store), Story 5-3 (sliders)

## Acceptance Criteria

**AC-1: Export with Non-default Weights (FR50)**
Given I have adjusted any weight to non-default,
When I export the architecture as YAML,
Then the exported file includes a weight_profile section with all 7 category weights.

**AC-2: Export Omits Default Weights (FR72)**
Given all weights are at default (1.0),
When I export,
Then the weight_profile section is OMITTED entirely.

**AC-3: Import v2 YAML with Weight Profile**
Given I import a v2 YAML with weight_profile,
When the architecture loads,
Then sliders set to imported values and dashboard reflects those weights.

**AC-4: Import v1 YAML -- Graceful Fallback**
Given I import a v1 YAML (no weight_profile),
When the architecture loads,
Then sliders at default 1.0, unweighted scores displayed.

**AC-5: Import Invalid Weight Profile -- Rejection (FR71)**
Given a v2 YAML with invalid weight_profile (value=2.0, unknown category, missing category),
When I import,
Then import fails with clear validation error message.

## Architectural Acceptance Criteria

**AC-ARCH-PATTERN-1:** Conditional export uses `isDefaultWeightProfile()` helper -- not inline comparison
**AC-ARCH-PATTERN-2:** Weight extraction happens inside existing yamlImporter.ts pipeline -- not a separate step
**AC-ARCH-PATTERN-3:** loadArchitecture accepts optional weightProfile; sets DEFAULT when absent
**AC-ARCH-PATTERN-4:** Semantic validation: all-zero weights normalized to equal weights with console warning
**AC-ARCH-NO-1:** Weight data NOT stored in Firestore -- per-architecture, per-YAML-file only
**AC-ARCH-NO-2:** No separate import function for weights -- extends existing pipeline
**AC-ARCH-NO-3:** Do NOT silently ignore invalid weight profiles -- surface validation error
