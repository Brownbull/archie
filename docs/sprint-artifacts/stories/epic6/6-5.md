## Overview

As a developer,
I want E2E tests covering the constraint guardrail flow,
So that regressions in constraint CRUD, violation display, navigation, and YAML round-trip are caught automatically.

**FRs:** Validates FR51-FR57, FR71-FR72 end-to-end

**Depends on:** Story 6-3 (UI), Story 6-4 (export/import)

## Acceptance Criteria

**AC-1: Add Constraint and See Violation**
Given authenticated user with architecture loaded,
When E2E test adds a constraint via the constraint panel,
Then violation badge appears on affected canvas nodes.

**AC-2: Click Violation Navigates to Component**
Given constraint violations visible,
When clicking a violation in the dashboard list,
Then canvas pans to the offending component.

**AC-3: Remove Constraint Clears Violations**
Given active constraint with violations,
When constraint is removed,
Then violation badges disappear from canvas.

**AC-4: Export/Import Round-Trip Preserves Constraints**
Given constraints defined,
When architecture exported and reimported,
Then constraints list matches and violations are re-evaluated.
