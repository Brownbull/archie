# Story 6-3: Constraint Management UI & Canvas Violation Indicators

## Status: drafted
## Epic: Epic 6 -- Constraint Guardrails

## Intent
**Epic Handle:** "Your red lines, visible on the canvas"
**Story Handle:** "This story paints the red lines on the canvas and gives users the control panel to draw them"

## Overview

As a user,
I want to add/edit/remove constraint guardrails from the dashboard and see violations highlighted on the canvas,
So that I can define my requirements and immediately see which components breach them.

**FRs:** FR51, FR53, FR54, FR55, FR56
**Size:** MEDIUM (5 pts) -- 5 tasks, ~22 subtasks, 7 files
**Risk Flags:** PURE_COMPONENT, CROSS_STORE, ERROR_RESILIENCE, E2E_TESTING
**Depends on:** Story 6-1 (types), Story 6-2 (engine + store)

## Acceptance Criteria

**AC-1: Constraint Management Panel (FR51, FR56)**
Given the dashboard expanded overlay is open,
When viewing the constraints section,
Then I can add a new constraint (select category, operator, threshold, label),
And I can edit or remove existing constraints.

**AC-2: Canvas Violation Indicators (FR53)**
Given active constraint violations exist,
When viewing the canvas,
Then affected nodes show a violation badge (warning icon) distinct from heatmap colors,
And the badge shows violation count for that node.

**AC-3: Dashboard Constraint Status (FR54)**
Given constraints are active,
When viewing the dashboard,
Then an aggregate status shows: "N constraints, M violations" (or "All clear" when M=0).

**AC-4: Click-to-Navigate (FR55)**
Given a constraint violation is listed in the dashboard,
When I click the violation entry,
Then the canvas pans and zooms to the offending component,
And the component's inspector opens.

**AC-5: Overlay-Only Visual (FR53)**
Given constraint violation indicators on the canvas,
When violations are shown,
Then they are a SEPARATE visual layer from heatmap (badge/icon overlay, not color change),
And removing all constraints removes all violation indicators.

## Architectural Acceptance Criteria

**AC-ARCH-LOC-1:** ConstraintPanel component at `src/components/dashboard/ConstraintPanel.tsx`

**AC-ARCH-NO-3:** No constraint management controls in compact DashboardPanel

## Dev Notes

**Click-to-navigate pattern:** React Flow provides `fitView({ nodes: [nodeId] })` and `setCenter(x, y, { zoom })`. Use the node's position to center. Then dispatch uiStore action to open inspector for that node.

**Badge positioning:** Use CSS `position: absolute; top: -8px; right: -8px;` within the node wrapper. React Flow custom nodes support arbitrary React content.

**Form UX:** Inline form within the panel (not a modal). Category dropdown populated from METRIC_CATEGORIES. Threshold dropdown shows "Low (1) / Medium (2) / High (3)". Label is optional -- auto-generate "{Category} {operator} {threshold}" if empty.
