# Story 10-3: E2E — Canvas Metrics & Blueprint Loading

## Status: done
## Epic: Epic 10 -- Canvas & Content Enhancements

## Intent
**Epic Handle:** "More to see, more to start from"
**Story Handle:** "This story proves inline metrics show the right data and blueprints load correctly — V5 + V6 compliance"

## Overview

As a user,
I want end-to-end proof that inline canvas metrics are weight-responsive and blueprints load correctly,
So that I can trust the visual feedback on the canvas matches the underlying data.

**FRs:** FR86, FR87, FR88
**Size:** SMALL (2 pts) -- 3 tasks, ~10 subtasks, 2 files
**Risk Flags:** E2E_TESTING
**Depends on:** Story 10-1 (inline metrics), Story 10-2 (blueprints)
**Type:** Hardening

## Acceptance Criteria

**AC-1: Inline Metrics Visible on Canvas**
Given components placed on the canvas,
When the canvas renders,
Then each component card shows 2 compact metric bars and the active variant name.

**AC-2: Metrics Respond to Weight Changes (V6: Semantic Verification)**
Given the current weight profile with performance=1.0 and cost=0.2,
When the user changes cost weight to 1.0 and performance to 0.2,
Then the inline metrics on at least one card change to show Cost instead of Perf as one of the top 2 metrics. (Verifies CONTENT, not just that something changed.)

**AC-3: Blueprint Loading**
Given 10 blueprints in the library,
When the user loads a blueprint onto the canvas,
Then all components appear with correct positions, connections, and inline metrics visible.

**AC-4: Demand Scenario + Inline Metrics**
Given an active demand scenario,
When inline metrics are displayed,
Then the values shown match the demand-adjusted values (not base values). Selecting "No Scenario" reverts them.
