# Story 10-1: Inline Canvas Metrics — Weight-Responsive Node Display

## Status: done
## Epic: Epic 10 -- Canvas & Content Enhancements

## Intent
**Epic Handle:** "More to see, more to start from"
**Story Handle:** "This story puts the most important metrics right on the canvas card — no click required to see what matters"

## Overview

As a user,
I want to see the top 2 metrics (by my current weight profile) and the active configuration variant name directly on each canvas component card,
So that I can understand trade-offs at a glance without clicking every component to open the inspector.

**FRs:** FR86, FR87, FR93
**Size:** MEDIUM (5 pts) -- 5 tasks, ~21 subtasks, 6 files
**Risk Flags:** PURE_COMPONENT, CROSS_STORE
**Depends on:** None (uses existing weight profile and component data)

## Acceptance Criteria

**AC-1: Active Variant Name on Card (FR86)**
Given a component on the canvas with a selected configuration variant,
When the card renders,
Then the active variant name is displayed below the component name in smaller text.

**AC-2: Top 2 Metrics by Weight (FR87)**
Given the current weight profile,
When a canvas card renders,
Then the 2 metric categories with the highest weight are shown as compact metric bars (name + numeric value + small bar) on the card.

**AC-3: Weight-Responsive Updates (FR87)**
Given the weight profile changes (via slider or scenario selection),
When the canvas re-renders,
Then the displayed metrics on each card update to reflect the new top 2 weighted categories. Cards may show different metrics than before if weights changed.

**AC-4: Demand-Adjusted Values (FR87)**
Given an active demand scenario,
When inline metrics are displayed,
Then they show demand-adjusted values (not base values), matching what the inspector would show.

**AC-5: Compact Layout**
Given the existing card dimensions,
When inline metrics are added,
Then the card grows vertically by no more than 40px. Metric bars use a condensed format (category abbreviation + value + mini bar).

**AC-6: No Metric Display When No Components**
Given a newly placed component with no metrics yet (edge case: library data not loaded),
When the card renders,
Then it shows the component name and variant only — no broken metric bars.

**AC-7: Constraint Warning Badges on Canvas Nodes (FR93)**
Given a component on the canvas with one or more active constraint violations,
When the canvas renders,
Then a constraint violation badge (existing `ConstraintViolationBadge` component) is visible on the node card, showing the count of active violations.

**AC-8: Badge Updates on Constraint Changes (FR93)**
Given constraints are added, modified, or removed,
When the constraint evaluator re-runs,
Then canvas badges update to reflect the current violation state without requiring manual refresh.

**AC-9: Badge and Inline Metrics Coexistence**
Given a node with both inline metrics (from AC-1/AC-2) and constraint violations,
When the card renders,
Then the badge appears in its existing position (top-right corner) without overlapping inline metric bars. Both are simultaneously visible and readable.
