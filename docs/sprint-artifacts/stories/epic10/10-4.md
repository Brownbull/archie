# Story 10-4: Architecture Report Export

## Status: drafted
## Epic: Epic 10 -- Canvas & Content Enhancements

## Intent
**Epic Handle:** "More to see, more to start from"
**Story Handle:** "This story turns a living canvas into a shareable snapshot -- the architecture speaks on paper"

## Overview

As a user,
I want to download a comprehensive markdown report of my current architecture assessment,
So that I can share my architecture analysis with stakeholders or save it for future reference.

**FRs:** FR94
**Size:** MEDIUM (4 pts) -- 4 tasks, 18 subtasks, 5 files
**Risk Flags:** DATA_PIPELINE, PURE_COMPONENT
**Depends on:** Story 10-1 (inline metrics exist -- report reads same data)

## Acceptance Criteria

**AC-1: Report Generation (FR94)**
Given an architecture with 3+ components on the canvas,
When the user clicks "Export Report" from the toolbar/menu,
Then a `.md` file is generated and downloaded containing a structured architecture assessment.

**AC-2: Report Content -- Executive Summary (FR94)**
Given a generated report,
When the executive summary section is read,
Then it includes: architecture name, component count, connection count, active tier, overall architecture score, active demand/failure scenarios (if any).

**AC-3: Report Content -- Per-Component Scores (FR94)**
Given a generated report,
When the component details section is read,
Then each component lists: name, category, active variant, top 3 weighted metric scores, heatmap status (healthy/warning/bottleneck).

**AC-4: Report Content -- Constraint Violations (FR94)**
Given a generated report with active constraints,
When the constraint section is read,
Then it lists all active violations: constraint label, affected node, actual score vs. threshold.

**AC-5: Report Content -- Scenario Comparison (FR94)**
Given a generated report with an active demand or failure scenario,
When the scenario section is read,
Then it shows baseline vs. demand-adjusted scores for the top 5 most-degraded components.

**AC-6: Performance (NFR25)**
Given an architecture with 15 components,
When report generation runs,
Then the complete report is generated and download-triggered within 3 seconds.

**AC-7: Filename Sanitization (Security -- INPUT_SANITIZATION)**
Given the architecture name contains special characters or path separators,
When the report filename is generated,
Then the filename is sanitized to contain only alphanumeric characters, hyphens, and underscores. No path traversal or injection is possible.

**AC-8: Data Provenance Footer (V2: Honest Uncertainty)**
Given the generated report,
When the footer section is read,
Then it includes a provenance statement: data is AI-generated and directional (not benchmarked); demand simulation values are estimates; metric scores reflect relative trade-offs, not absolute performance. This is the first artifact that leaves the tool — stakeholders reading the report have no in-app context.

## Dev Notes

**Pure function approach:** The report generator takes a fully-resolved `ReportData` object and returns a markdown string. No store access inside the generator -- the button component collects all data and passes it in. This makes the generator trivially testable.

**Filename sanitization (from security review):** Architecture names come from user input (YAML import or manual entry). The filename must be sanitized before use in the download anchor:
```typescript
const sanitize = (name: string): string =>
  name.replace(/[^a-zA-Z0-9\s-]/g, '').replace(/\s+/g, '-').substring(0, 100);
```
