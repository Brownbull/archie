# Story 7-4: E2E -- Data Context & Fit Analysis Flow

## Status: drafted
## Epic: Epic 7 -- Data Context Items

## Intent
**Epic Handle:** "Your data, scored against every component"
**Story Handle:** "This story validates the complete data context and fit analysis experience end-to-end"

## Overview

As a developer,
I want E2E tests covering the data context item flow,
So that regressions in item CRUD, fit indicators, variant-sensitive updates, and YAML round-trip are caught automatically.

**FRs:** Validates FR58-FR63, FR71-FR72 end-to-end
**Size:** SMALL (2 pts) -- 2 tasks, ~11 subtasks, 2 files
**Risk Flags:** E2E_TESTING
**Depends on:** Story 7-2 (UI), Story 7-3 (export/import)

## Acceptance Criteria

**AC-1: Add Data Item and See Fit**
Given authenticated user with architecture loaded and a component selected,
When E2E test adds a data context item via the inspector,
Then fit indicator appears with a level badge.

**AC-2: Variant Switch Updates Fit**
Given data context item on a component,
When the component's variant is switched,
Then fit indicator updates to reflect the new variant.

**AC-3: Expand Fit Explanation**
Given a fit indicator displayed,
When clicked,
Then explanation and contributing factors are visible.

**AC-4: Export/Import Round-Trip Preserves Data Context**
Given data context items defined,
When architecture exported and reimported,
Then items are restored and fit indicators re-derived.

## Architectural Acceptance Criteria

**AC-ARCH-LOC-1:** Spec at `tests/e2e/data-context.spec.ts`
**AC-ARCH-PATTERN-1:** SCREENSHOT_DIR constant, data-testid selectors, getByRole fallback
**AC-ARCH-PATTERN-2:** Auth pre-loaded from storageState
**AC-ARCH-PATTERN-3:** Screenshots to `test-results/data-context/`
**AC-ARCH-NO-1:** No waitForTimeout > 1000ms
**AC-ARCH-NO-2:** No browser.newContext() for authenticated tests

## Dev Notes

**Form interaction in Playwright:** Standard form selectors -- name input, access pattern dropdown, size dropdown, structure type dropdown. All identifiable via data-testid attributes.

**Variant-dependent fit:** The test asserts that the fit indicator re-renders after variant switch. The actual fit level may or may not change depending on test data -- the test just verifies reactivity.
