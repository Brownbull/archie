# Story 7.5-3: Pathway Guidance UI — Badge Hint & Dashboard Section

## Status: ready-for-dev
## Epic: Epic 7.5 -- Weighted Pathway Guidance

## Intent
**Epic Handle:** "Show the roads, not just the dot"
**Story Handle:** "This story puts the roads in front of the user -- a hint on the tier badge and full details in the dashboard"

## Overview

As a user,
I want to see a brief suggestion count on the tier badge and a detailed pathway guidance section in the Dashboard Overlay,
So that I discover growth opportunities without leaving the canvas and explore them in full when ready.

**FRs:** FR78, FR79
**Size:** MEDIUM (5 pts) -- 4 tasks, ~14 subtasks, 5 files
**Risk Flags:** PURE_COMPONENT, ERROR_RESILIENCE
**Depends on:** Story 7.5-2 (usePathwaySuggestions hook must exist)

## Acceptance Criteria

**AC-1: Tier Badge Hint (FR79)**
Given pathway suggestions exist (suggestions.length > 0),
When the tier badge popover is open,
Then below the gap list, a link shows "N suggestions" that opens the Dashboard Overlay to the Pathway Guidance section.

**AC-2: No Hint at Max Tier**
Given the architecture is at max tier (no gaps),
When the tier badge popover is open,
Then no suggestion link appears.

**AC-3: Dashboard Overlay Section (FR78)**
Given the Dashboard Overlay is open,
When suggestions exist,
Then a "Pathway Guidance" collapsible section appears after Constraint Guardrails, with a badge showing suggestion count.

**AC-4: Suggestion Card Content**
Given a suggestion in the Pathway Guidance section,
When displayed,
Then it shows: component name, category, which gap it addresses, weighted score, constraint safety indicator (safe/warning), and data context fit level (when available).

**AC-5: Empty State**
Given no suggestions (max tier or no components),
When the Pathway Guidance section is opened,
Then it shows an appropriate empty message.

**AC-6: Overlay Scroll Target**
Given the tier badge link is clicked,
When the Dashboard Overlay opens,
Then the Pathway Guidance section is expanded and scrolled into view.

## Architectural Acceptance Criteria

**AC-ARCH-LOC-1:** Pathway panel in `src/components/dashboard/PathwayGuidancePanel.tsx` (CREATE)
**AC-ARCH-LOC-2:** TierBadge modifications in `src/components/dashboard/TierBadge.tsx` (MODIFY)
**AC-ARCH-LOC-3:** DashboardOverlay modifications in `src/components/dashboard/DashboardOverlay.tsx` (MODIFY)
**AC-ARCH-PATTERN-1:** PathwayGuidancePanel consumes `usePathwaySuggestions` hook -- no direct engine calls
**AC-ARCH-PATTERN-2:** Suggestion cards are read-only display -- no mutation actions

**AC-ARCH-NO-2:** No new state in architectureStore for UI display
**AC-ARCH-NO-3:** No `dangerouslySetInnerHTML` -- all suggestion text is React-escaped
