## Acceptance Criteria

**AC-1: Full Flow — Suggestions Visible**
Given an authenticated user with an architecture at Tier 1 (Foundation),
When they open the tier badge popover,
Then they see a "N suggestions" link, and clicking it opens the Dashboard Overlay Pathway Guidance section with ranked suggestion cards.

**AC-2: Weight Change Affects Ranking**
Given the Pathway Guidance section is visible with suggestions,
When the user changes a priority weight slider (e.g., scalability to high),
Then suggestion ranking reorders to prioritize scalability-strong components.

**AC-3: Constraint Warning Visible**
Given the user has an active constraint (e.g., cost <= 7.0),
When a suggestion's default variant exceeds that threshold,
Then the suggestion card shows a constraint warning badge.

**AC-4: Max Tier — No Suggestions**
Given an architecture at max tier,
When the tier badge popover is open,
Then no suggestion link appears, and the Pathway Guidance section shows an empty state.

**AC-5: V4 Drift Resolution Verification**
Given the complete E2E flow passes,
When reviewed against V4 ("Show the roads, not just the dot"),
Then the E2E proves: users see actionable, personalized growth paths under their weight profile.
