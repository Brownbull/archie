# Story 7.5-1: Tier Gap Analysis Engine & Pathway Suggestions

## Status: done
## Epic: Epic 7.5 -- Weighted Pathway Guidance

## Intent
**Epic Handle:** "Show the roads, not just the dot"
**Story Handle:** "This story teaches Archie which components to suggest and why -- the pure engine that turns gaps into ranked suggestions"

## Overview

As a user,
I want the system to analyze tier gaps and produce a ranked list of component suggestions that account for my weight profile, constraints, and data context,
So that I see actionable next steps instead of abstract gap descriptions.

**FRs:** FR73, FR74, FR75, FR76, FR77, NFR20
**Size:** MEDIUM (5 pts) -- 4 tasks, ~16 subtasks, 4 files
**Risk Flags:** PURE_COMPONENT, DATA_PIPELINE
**Depends on:** None (uses existing tier evaluation output + component library data)

## Acceptance Criteria

**AC-1: Gap-to-Suggestion Mapping (FR73, FR74)**
Given tier gaps from `evaluateTier()`,
When `computePathwaySuggestions()` is called with the component library,
Then each gap produces a list of candidate components that would close it, excluding components already on the canvas.

**AC-2: Weight-Aware Ranking (FR75)**
Given a non-default weight profile (e.g., scalability=1.0, cost=0.2),
When suggestions are ranked,
Then components with higher scores in heavily-weighted categories appear first.

**AC-3: Constraint Heuristic Filter (FR76)**
Given active constraints (e.g., cost must not exceed 7.0),
When a candidate's default variant cost metric exceeds 7.0,
Then the suggestion is flagged `isConstraintSafe: false` with the constraint name as warning.

**AC-4: Data Context Fit Scoring (FR77)**
Given data context items on the architecture,
When suggestions are scored,
Then each candidate includes a `fitLevel` from the existing `evaluateFit()` logic. When no data context items exist, `fitLevel` is undefined.

**AC-5: Performance (NFR20)**
Given 30 candidate components and 6 tier gaps,
When suggestions are computed,
Then results complete within 50ms.

**AC-6: Deduplication**
Given the same component suggested by multiple gaps,
When results are assembled,
Then duplicate suggestions are merged with the most specific gap as primary reason and other gaps as secondary.
