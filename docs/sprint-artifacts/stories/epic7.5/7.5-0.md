## Dev Notes

**Context for brainstorming:**
- The tier system currently recalculates with weighted scores but shows no growth path
- A user who sets scalability=high, reliability=high doesn't see "adding a cache would move you from Tier 2 to Tier 3 under your priorities"
- Three foundation layers are now complete:
  - Weights (Epic 5): rank suggestions by user priorities
  - Constraints (Epic 6): filter suggestions that violate red lines
  - Data context (Epic 7): fit suggestions to data patterns

**Key questions to resolve:**
1. WHERE does pathway guidance appear? (tier overlay extension vs. dashboard section vs. inspector panel vs. new panel)
2. HOW does the gap analysis work? (component library gap detection, metric deficit calculation)
3. HOW are suggestions ranked? (weighted scoring of potential improvements)
4. WHAT granularity? (component-level suggestions vs. configuration-level vs. both)
5. HOW does it interact with existing progressive disclosure? (collapsed hints vs. expandable recommendations)

**Architecture reference:**
- `src/engine/recalculationEngine.ts` -- pure metric recalculation
- `src/engine/constraintEvaluator.ts` -- pure constraint evaluation
- `src/engine/fitEvaluator.ts` -- pure fit evaluation
- `src/stores/architectureStore.ts` -- tier evaluation logic
- `_bmad-output/planning-artifacts/architecture.md` -- canonical architecture doc
